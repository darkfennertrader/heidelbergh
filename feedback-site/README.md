Static feedback form served via **CloudFront + S3** with submissions sent to email via **AWS SES**.

## Architecture

```
clinician browser
      │ HTTPS
      ▼
CloudFront (*.cloudfront.net, free TLS)
      │
      ▼
S3 bucket (private, OAC-locked)   ← index.html, style.css, app.js, config.js
      │
      │  POST /feedback (JSON)
      ▼
API Gateway HTTP API
      │
      ▼
Lambda (Python 3.12, 128 MB)
      │
      ▼
SES → darkfenner69@gmail.com, dottpaolomilani@hotmail.com
```

## Prerequisites (one-time)

### 1. IAM permissions

Attach the inline policy `AppWayFeedbackSiteDeploy` to the EC2 role `EC2AppWayBackendRole`.
The JSON is at [`../docs/iam-feedback-deploy-policy.json`](../docs/iam-feedback-deploy-policy.json).

Steps:
1. AWS Console → IAM → Roles → `EC2AppWayBackendRole`
2. Permissions tab → Add permissions ▾ → Create inline policy
3. JSON tab → paste the contents of `iam-feedback-deploy-policy.json`
4. Name: `AppWayFeedbackSiteDeploy` → Create policy

### 2. SES verified identities (eu-west-1)

Both sender and recipient addresses must be verified while in SES sandbox:

```bash
aws ses verify-email-identity --email-address darkfenner69@gmail.com    --region eu-west-1
aws ses verify-email-identity --email-address dottpaolomilani@hotmail.com --region eu-west-1
```

Click the verification link AWS emails to each address.

## Deploy

```bash
cd feedback-site
chmod +x deploy.sh
./deploy.sh
```

The script:
1. Creates/verifies the deploy S3 bucket.
2. Zips the Lambda and uploads it.
3. Runs `aws cloudformation deploy` (creates or updates the stack).
4. Generates `public/config.js` with the API Gateway URL.
5. Syncs `public/` to the site S3 bucket.
6. Invalidates the CloudFront cache.
7. Prints the **CloudFront URL** — paste this into Heyex.

### First deploy (~5 min)

CloudFront distributions take 5–10 min to deploy globally on first create.
Subsequent updates are near-instant (cache invalidation takes ~30 sec).

## Update site content only

To push HTML/CSS/JS changes without touching infrastructure:

```bash
# Regenerate config.js if needed (only if API URL changed)
API_URL=$(aws cloudformation describe-stacks --stack-name appway-feedback \
  --region eu-west-1 --query 'Stacks[0].Outputs[?OutputKey==`ApiURL`].OutputValue' --output text)
echo "window.FEEDBACK_API_URL = \"${API_URL}\";" > public/config.js

aws s3 sync public/ s3://$(aws cloudformation describe-stacks --stack-name appway-feedback \
  --region eu-west-1 --query 'Stacks[0].Outputs[?OutputKey==`SiteBucket`].OutputValue' \
  --output text)/ --delete --region eu-west-1

aws cloudfront create-invalidation \
  --distribution-id $(aws cloudformation describe-stacks --stack-name appway-feedback \
    --region eu-west-1 --query 'Stacks[0].Outputs[?OutputKey==`DistributionId`].OutputValue' \
    --output text) \
  --paths "/*"
```

## Tear down

```bash
# Empty the site bucket first (CloudFormation can't delete non-empty buckets)
SITE_BUCKET=$(aws cloudformation describe-stacks --stack-name appway-feedback \
  --region eu-west-1 --query 'Stacks[0].Outputs[?OutputKey==`SiteBucket`].OutputValue' --output text)
aws s3 rm "s3://${SITE_BUCKET}/" --recursive --region eu-west-1

aws cloudformation delete-stack --stack-name appway-feedback --region eu-west-1
```
